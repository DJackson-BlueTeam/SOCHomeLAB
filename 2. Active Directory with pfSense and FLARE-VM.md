# Complete SOC Lab Procedure: Windows Server & Active Directory Setup

## Phase 1: Virtual Box VM Creation

**1. Create New VM**

**2. Name & OS**
-  Name: SOC-DC01 (What is desirable)
- Type: `Microsoft Windows`` | Version: Windows 2022 (64-bit)` (or latest version)

**3. The "Unattended" Fix (Crucial): Uncheck the box "Skip Unattended Installation"**
- This prevents VirtualBox from trying to automate the install, which causes the "License Terms" error.

**4. Hardware**
- RAM: At least 2048 MB (4096 MB is better if your host PC allows).
-  Processors: 2 CPUs

**5. Hard Disk**
    - Create a virtual disk of at least 50 GB (Desktop Experience needs more space than core)

##Phase 2: Networking Setup (The "Two-Door-System")
- We learned that pfSense expects `WAN` on `Adapter 1` and `LAN` on `Adapter 2`. We must match this on the Server.

1. Adapter 1 (WAN)
- Set to Bridge Adapter (Select your physical Wi-FI/Ethernet card)
  - If having a connection issue later, switch to NAT.
2. Aapter 2 (LAN)
- Enable and set to Internal Network (Name:`intnet`)
  - This "plugs" the server into the same virtual switch as the pfSense LAN and FLARE-VM
 
## Phase 3: Windows Installation (Desktop)
Avoiding the "Black Screen" (Server Core) by selecting the correct graphical version.

**1. Start the VM and boot from the Windows Server ISO**

**2. Operating System Selection**
   - Choose Windows Server 2022 Standard Evaluation (Desktop Experience)
       - Warning: If the "Desktop Experience" is not selected, the Windows Server Command Prompt will be displayed only.

**3. Installation Type**
   - Select "Custom: Installation Microsoft Server Operating System only (advanced)"

**4. Partioning**
   - If seeing old partions from a failed attempt, Delete all of them until you have one "Unallocated Space 50GB", then click next.

**5. The Reboot Loop Fix**
   - Once the installer finishes and tries to restart, Shut down the VM. Go to `Settings > Storage`, right click the ISO (blue disk icon), and select Remove Disk from Virtual Drive.
     - This prevents the VM from booting back into the installer.
