# Building a Secure SOC Sandbox: Network Isolation & Intergration Guide

### This guide outlines constructing a multi-node SOC Home Lab designed for malware analysis and security monitoring. The primary objective is to create a "Sandbox" environment where malicious traffic is contained within a private virtual network, monitored by a SIEM, and prevented from reaching the host machine or the real-world internet.

## Lab Architecture Overview

- The lab utilizes pfSense as a virtual firewall to segment the "Sandbox" from the prodcution (Home) network.

Phase 1: pfSense - The Perimeter Foundation
---
pfSense acts as the "Gatekeeper". It is the only VM that is allowed to talk to the actuall internet source.

**1. VirtaulBox Network Configuration**
- Adapter 1 (WAN): Set to Bridged or NAT. provides the internet source
- Adapter 2 (LAN): Set to Internal Network (Name: `Intnet`). This act as the isolation switch for the lab.

**2. Initial Setup**
- Assign `em0` to WAN and `em1` to LAN
- Configure the LAN IP to `10.0.0.1` with a `/24` (255.255.255.0) subnet mask.

**3. DHCP Service**
- Enable DHCP on the LAN interface to allow new VMs to recieve an initial IP before setting them to static.

Phase 2: Windows SOC-DC (Optional Implementation)
---
If simulating an enterprise enviornment, the Domain COntroller provides centralized identity.

**1. Network Cofiguration**
- Adapter 1: Set to Internet Network `Intnet`

**2. Static IP Assignment**
- IP: `10.0.0.2` | Gateway: `10.0.0.1`
- DNS: `127.0.0.1` (The server must resolve its own Active Directory names).

**3. Firewall Hardening**
- Disbale the WIndpws Firewall for the Domain, Private and Public profiles to prevent interface with initial domain joins.

Phase 3: FLARE-VM The Analysis Workbench 
---
The FLARE-VM is where malware is executed. It must be strictly isolated.

**1. Network COnfiguration**
- Adapter 1: Set to Internet Network `Intnet`

**2. Static IP Assignment**
- IP: 10.0.0.10 | Gateway: `10.0.0.1`

**3. DNS Strategy (Critical)**
- For Upfates: Set DNS to `10.0.0.1` (pfSense).
- For Analysis: Set DNS to `10.0.0.3` (REMnux). This forces all maliciouus traffic into the network listener.

Phase 4: REMnux - The Network Simulator
---
REMnux provides "Fake Internet" services so malware thinks it is online while remaining trapped.

**1. Network COnfiguration**
- Adapter 1: Set to Internal Network `Intnet`

**2. Static Assignment (Netplan)**
- Configure `/etc/netplan/00-intstall-config.yaml` with IP 10.0.0.3 and Gateway `10.0.0.1`.

**3. INetSim Intergration**
- Start INetSim (`sudo inetsim`). This will now listen for any traffic coming from the FLARE-VM and provide fake HTTO,DNS and STMP responses.
