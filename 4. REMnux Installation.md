# REMnux Installation: Intercepting Malware Traffic.
## Procedure: Insalling and Intergrating REMnux for Malware Analysis

Phase 1: VirtualBox Deployment
---

**1. Download Virtual Appliance**
- Obtain the `.ova` file from: the https://docs.remnux.org/install-distro/get-virtual-appliance

2.**Import to VirutalBox**
- Go to `File > Import Appliance` and select the `.ova` file
- RAM: Assign at least 2048 MB (2 GB)

**3. Network Settings (The "Sandbox" Connection)**
- Right-click the `REMnuxVM > Setting > Network`.
- Adapter 1: Set to Internal Network `intnet`(pfSense Firewall)
- Crucial: Ensure Adapter 2, 3, and 4 are Disabled to prevent the VM from bypassing the firewall.

Phase 2: Static IP Configuration
---
Linux Network Configuration is sensitive to formatting. Please follow these steps to set the Static IP.

**1. Identifying the Interface Name**
- Open the terminal and type `ip a`. Look for the name of the second entry (usually `enp0s3`)

**2. Edit the configuration file**
- Copy and paste this command in the REMnux terminal shown below
```sudo nano /etc/netplan/00-installer-config.yaml```

**3. Apply the Configuration**
- Delete the existing text and replace it with the following. (Use `TAB` for consistent indentation.)

```
   network:
    version: 2
    renderer: networkd
    ethernets:
      enp0s3:
   #replace this with the actually IP
      addresses:
        - 123.456.1.3/24 
      nameservers:
        addresses: [123.456.1.2, 8.8.8.8]
      routes:
        - to: default
          via: 123.456.1.1
```
**4. Secure and Apply**
- Run these commands to set correct permissions and activate the settings

``` sudo netplan apply ```

Phase 3: Intergration & Network Simulation
---
Here, we will be turning REMnux to a FAKE INTERNET for the malware to target instead of the ACTUALLY INTERNET (Which is protected by the pfSense firewall)

**1. Start INetSim**
```sudo inetsim```

**2. FLARE-VM Configuration**
- On the FLARE-VM, chagne the preferred DNS to `123.456.1.3` (Will be broken down step by step in a All in one Network Configuration file for pfSense, FLARE-VM, REMnux and Ubuntu for WAZUH).

**3. Verification**
- Go to yhe FLARE-VM, open a browser, and attempt to naivagte to anywebsite.
- If configured correctly, you will see the INetSim default landing page instead of the real internet.

Phase 4: Troubleshooting 
---
Following issues may be encountered

**1. YAML Indentation Error**
- YAML files requires exact spacing. If an "incosistent indentation" error occurs, verify that every level is indented with exaclty two space (or correct tabing)

**2. Interface Naming**
- Modern Linux distribution often uses `enp0s3` instead of eth0.
- Always verify the name with `ip a` before editing the config.

**4. Gateway4 Deprecation**
- Newer versions of Netplan prefer the `routes` format. The configuration above uses the updates standard to ensure a clean boot without warnings.

**5. Bypassing The Firewall**
- If REMnux can still ping Google while Pfsenes WAN is disabled, check for a second adpater enabled in the VirtualBox and disable it.
