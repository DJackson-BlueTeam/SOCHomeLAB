# FLARE-VM: The Installation Procedure 

This guide provides end-to-end process for building a FLARE-VM malware analysis lab. The Host Machine of all of the VM OS must have a minimum of 16 GB RAM and an additional - up to - 500 GB to 1 TB of SSD.
https://www.youtube.com/watch?v=wYBgAb0rw1k <--- Basic VM/Windows Installation

Phase 1: Initial Windows VM Creation
---
**1. Download Windows ISO**
- Obtain a Windows 10 Pro or 11 ISO (If downloaded windows 10 home, it can be updated virtually shown in the later phase.)
https://www.microsoft.com/en-us/software-download/windows10?msockid=14411f7e24d963171a8309ea258b6220

**2. VirtualBox Settings**
- Name: (A name tailored to Malware Lab)
- Base Memory: At least 4096 MB (4GB)
- Processor: At least 2 CPUs.
- Hard Disk: Create a Virtual Hard Disk of 100 GB (Dynamically Allocated).

**3. Installing Windows**
- Walk through a standard windows installing. When prompted for a user, use a simple name `Malware Lab` or `SOC Lab` or anything in that aspect.

Phase 2: Drivers Guest Addtions (Only if not automatically installed with the VM Box)
---

**1. Install Guest Additions**
- In the VM windows menu: `Device > Insert Guest Additions CD Image`
- Open File Explorer inside the Windows VM. Find CD Drive.
- Right-click `VBoxWindowsAdditions-amd64` and select run as Administrator.
- Follow the prompt and reboot the VM.

**2. Enable Shared Clipboard**
- In the VM menu; `Device > Shared Clipboard >Bidirectional` (allows the user to copy and past from the host machine to the virtual machien or vice versa).

**3. Enable Group Policy (If using windows Home)**
- Right-click on the command Prompt and Run as Administrator.
- Copy and paste the script
- Then reboot the Windows VM Machine.

``Batch Script  for Command Prompt``

```
@echo off 
pushd "%~dp0" 
dir /b %SystemRoot%\servicing\Packages\Microsoft-Windows-GroupPolicy-ClientExtensions-Package~3*.mum >List.txt 
dir /b %SystemRoot%\servicing\Packages\Microsoft-Windows-GroupPolicy-ClientTools-Package~3*.mum >>List.txt 
for /f %%i in ('findstr /i . List.txt 2^>nul') do dism /online /norestart /add-package:"%SystemRoot%\servicing\Packages\%%i" 
pause
```

Phase 3: Manual Security Disabling 
---
**This must be completed before running the Nuclear Execution Script **

**1. Disable Tamper Protection**
- Go to `Windows Security > Virus & Threat Protection > Manage Setiings`
- Turn Tamper Protection OFF.

2. Disable Real-Time Protection
- Turn Real-Time-Protection OFF.

3. Disable Cloud Protection
- Turn `Cloud-Delivered Protection` and `Automatic Sample Submission` OFF.

Phase 4: The "Nuclear" Execution Script
---

This script nukes Windows Defender, the Windows Firewall, and all associated threat detection services that will INTERFERE with FLARE VM Machine installation.

**1. Open PowerShell as Administrator**
- Copy and paste the PowerShell Script.

    ```
    # 1. Create Registry Paths for Defender and Firewall
    $paths = @(
    "HKLM:\SOFTWARE\Policies\Microsoft\Windows Defender",
    "HKLM:\SOFTWARE\Policies\Microsoft\Windows Defender\Real-Time Protection",
    "HKLM:\SOFTWARE\Policies\Microsoft\WindowsFirewall\StandardProfile",
    "HKLM:\SOFTWARE\Policies\Microsoft\WindowsFirewall\DomainProfile",
    "HKLM:\SOFTWARE\Policies\Microsoft\WindowsFirewall\PublicProfile"
    )
    foreach($path in $paths) { if (!(Test-Path $path)) { New-Item -Path $path -Force } }

    # 2. Nuke Windows Defender
    Set-ItemProperty -Path "HKLM:\SOFTWARE\Policies\Microsoft\Windows Defender" -Name "DisableAntiSpyware" -Value 1 -Force
    Set-ItemProperty -Path "HKLM:\SOFTWARE\Policies\Microsoft\Windows Defender\Real-Time Protection" -Name "DisableRealtimeMonitoring" -Value 1 -Force
    Set-ItemProperty -Path "HKLM:\SOFTWARE\Policies\Microsoft\Windows Defender\Real-Time Protection" -Name "DisableBehaviorMonitoring" -Value 1 -Force

    # 3. Nuke Windows Firewall
    Set-ItemProperty -Path "HKLM:\SOFTWARE\Policies\Microsoft\WindowsFirewall\StandardProfile" -Name "EnableFirewall" -Value 0 -Force
    Set-ItemProperty -Path "HKLM:\SOFTWARE\Policies\Microsoft\WindowsFirewall\DomainProfile" -Name "EnableFirewall" -Value 0 -Force
    Set-ItemProperty -Path "HKLM:\SOFTWARE\Policies\Microsoft\WindowsFirewall\PublicProfile" -Name "EnableFirewall" -Value 0 -Force

    # 4. Disable Security Services (Ignore 'Access Denied' errors here)
    Set-Service -Name "SecurityHealthService" -StartupType Disabled -ErrorAction SilentlyContinue
    Set-Service -Name "wscsvc" -StartupType Disabled -ErrorAction SilentlyContinue
    
    # 5. Final Policy Force
    gpupdate /force```
