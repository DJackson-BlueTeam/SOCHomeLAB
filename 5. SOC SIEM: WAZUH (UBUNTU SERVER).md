# Building The SOC SIEM Foundation (Ubuntu Server)

Phase 1: VirtualBoc VM Configuration
---

The most common obstacle is using automated tools like multipass. For a professional lab, always use the manual ISO method.

**1. Download The ISO**
- Obtain the Ubuntu Server 22.04 LTS (or latest version) from the Ubuntu website.
- https://ubuntu.com/download/server

**2. Create New VM**
- Name: SOC-SIEM (What your heart desires)
- Type: `Linux` | Version: Ubuntu (64-bit)

**3. Hardware Allocation**
- RAM: 4096 MB (4 GB) minimum. Wazuh are resource heavy.
- Processor: 2 CPUs

**4. Networking**
- Adapter 1: set to Interneal Network (will be with the pfSense Firewall network)
- This ensures the SIEM is "Plugged in" to the same switch as the FLARE-VM and Domain COntroller (DC is optional for Corporate Experience Enterprise)

Phase 2: Static IP & Netplan Configuration
---
**1. Identify Interface**
- Run `ip a` and look for the name of the active link (`enp0s3`)

**2. Edit Config**
- Run ```sudo nano /etc/netplan/00-intstaller-config.yaml```

**3. YAML Template**
- use this script in yaml.

```
network:
  version: 2
  renderer: networkd
  ethernets:
    enp0s3:
      dhcp4: no
      addresses:
        - 123.456.1.5/24
      nameservers:
        addresses: [123.456.1.2, 8.8.8.8]
      routes:
        - to: default
          via: 123.456.1.1
```

**4. Apply Settings**
- `sudo netplan apply`
- ignore warnings or run ```sudo chmod 600 /etc/netplan/00-installer-config.yaml``` to fix warning permissions.

## Ubuntu Server Is Now Setup And Wazuh SIEM & EDR Can Be Installed

# Deploying Wazuh SIEM & EDR

Wazuh acts as both a Log Manager and a Threat Detector

Phase 1: Installation
---

**1. Enable Internet**
- TEMPORARILY change the VM Adapter 1 to NAT in Virtual BOX settings

**2. Run Installer**

```
curl -sO https://packages.wazuh.com/4.7/wazuh-install.sh
sudo bash wazuh-install.sh -a -i

# -i flag ignores hardware checks if you are running on limited RAM
```

**3. Save Credential**
- CRITICAL: At the end of the script, copy the admin password and store it. IT IS SHOWN ONLY ONCE

**4. Re-Isolate**
- Change the VM Adapter back to Internal Network ( May have to power off the server to conduct the adapter readjustments).

Phase 2: Deploying the "Eyes and Ears" (Wazuh Agent)
---

**1. Access Dashboard**
- On the FLARE-VM brows to the browser with the ip address of the Ubuntu Server `https://123.456.1.1`

**2. Generate Agent command**
- Go to `Agent > Deploy new agent`. Select Windows and enter the SIEM IP (123.456.1.5)

**3. Powershell Command**
- Once on the Wazuh dashboard, it should have a step-by-step task to follow on the FLARE-VM machine to start its active monitoring and Threat Detection.
- If having trouble still, copy and paste the command below in the windows PowerShell (MAKE SURE YOU RUN IS AS ADMINISTRATOR)

```
$msiPath = "$env:TEMP\wazuh-agent.msi"
Invoke-WebRequest -Uri "https://packages.wazuh.com/4.x/windows/wazuh-agent-4.7.5-1.msi" -OutFile $msiPath
msiexec.exe /i $msiPath /q WAZUH_MANAGER='123.456.1.5' WAZUH_AGENT_NAME='FLARE-VM' WAZUH_REGISTRATION_SERVER='123.456.1.5'
```

**4. Start Service**
- ```NET START WazuhSvc```
